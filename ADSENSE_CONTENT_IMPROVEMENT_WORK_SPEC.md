# AdSense 콘텐츠 가치 개선 작업 명세

작성일: 2026-05-23

## 1. 목적

코워크스페이스가 AdSense 심사에서 `가치가 별로 없는 콘텐츠` 사유로 거절된 상태를 개선한다.

이 작업의 목적은 단순히 글자 수를 늘리는 것이 아니라, 기능 중심 플랫폼에 맞는 문서형 콘텐츠를 보강해 다음을 충족하는 것이다.

- 각 도구가 해결하는 실제 업무 문제를 명확히 설명한다.
- 도구별로 서로 다른 사용 맥락과 활용 방식을 제공한다.
- 카테고리 페이지가 단순 링크 목록이 아니라 업무 흐름을 설명하게 한다.
- 브라우저 처리, 서버 저장 없음, 지원 형식, 한계, 검수 필요성을 신뢰 가능한 방식으로 안내한다.
- AdSense 재검토 전 사이트가 충분한 고유 가치와 사용자 경험을 갖췄는지 점검 가능하게 한다.

## 2. 현재 이슈

AdSense 거절 사유:

- 광고 게재가 준비되지 않은 사이트
- 정책 위반 발견
- 가치가 별로 없는 콘텐츠

확인된 전제:

- 사이트 소유권은 이미 확인된 상태다.
- 핵심 문제는 소유권이 아니라 콘텐츠 가치와 사용자 경험 부족으로 본다.
- 코워크스페이스는 블로그 사이트가 아니라 브라우저 기반 업무 도구 플랫폼이다.

## 3. 콘텐츠 전략

기능 중심 플랫폼에 억지 정보성 글을 붙이는 방식은 사용하지 않는다.

코워크스페이스의 콘텐츠는 다음 성격이어야 한다.

- 제품 문서
- 사용 시나리오
- 업무 흐름 안내
- 입력/결과 예시
- 처리 한계와 개인정보 안내

즉, "읽을거리"를 만드는 것이 아니라 "도구를 더 잘 쓰게 하는 설명"을 만든다.

## 4. 비범위

이번 개선에서 하지 않을 것:

- 메인 화면 구조를 광고 심사용 긴 문서 형태로 바꾸지 않는다.
- 모든 도구에 같은 FAQ를 반복하지 않는다.
- 도구와 관계없는 블로그 글을 대량 생산하지 않는다.
- 작업 영역, 입력창, 업로드 박스, 결과 영역 안에 광고나 긴 설명을 넣지 않는다.
- 사용자 입력, 파일명, 변환 결과, 녹음 내용 등을 분석 이벤트나 외부 API로 보내지 않는다.

## 5. 이미 완료된 작업

아래 작업은 이미 반영 및 배포되었다.

- 홈을 제외한 카테고리/도구 페이지의 기존 하단 `실제 사용 시나리오` 정적 블록 제거
- 도구 상세의 `사용 예시와 자주 묻는 질문` 구조 제거
- 한국어 도구 30개에 도구별 고유 `사용 시나리오` 3개씩 추가
- 영어 도구 30개에 도구별 고유 `Usage Scenarios` 3개씩 추가
- 일본어/중국어 도구별 시나리오 누락 여부를 감사 스크립트에서 검사
- FAQ 구조화 데이터 출력 중단
- 감사 스크립트가 도구별 시나리오 누락을 검사하도록 변경

관련 커밋:

- `8f4c0ca Remove tool scenario blocks`
- `ce4ab98 Add tool-specific usage scenarios`
- `c89d11e Add localized tool scenarios`

## 6. 다음 작업 범위

### 6.1 도구별 입력 예시 추가

각 도구에 실제 입력 예시를 추가한다.

예시:

- PDF 합치기: 견적서 PDF, 사업자등록증 PDF, 계약 첨부 PDF
- CSV Excel 변환: 주문 내역 CSV, 설문 응답 CSV, CP949 인코딩 CSV
- AI 텍스트 정리: Markdown 기호가 섞인 AI 답변, 보고서 초안, 메신저 공유 문안
- EXIF 제거: 휴대폰 사진, 위치정보가 포함된 JPG, 거래처 전달 이미지

목표:

- 사용자가 어떤 자료를 넣으면 되는지 즉시 이해한다.
- 도구별 콘텐츠가 서로 다르게 보인다.

### 6.2 도구별 결과 활용 방식 추가

각 도구의 결과물을 어디에 쓰는지 설명한다.

예시:

- PDF 합치기 결과: 이메일 제출, 행정 시스템 업로드, 회의자료 배포
- PDF 페이지 추출 결과: 검토용 일부 문서 공유, 증빙 페이지만 제출
- 이미지 압축 결과: 채용 사이트 업로드, 학교 제출, 쇼핑몰 이미지 등록
- 자막 변환 결과: YouTube 업로드, 웹 플레이어 적용, 편집툴 가져오기

목표:

- 도구가 단순 변환기가 아니라 실제 업무 흐름에 연결된다는 점을 보여준다.

### 6.3 도구별 주의점과 한계 보강

각 도구의 처리 조건과 한계를 도구별로 설명한다.

필수 포함 항목:

- 브라우저 처리 여부
- 자체 서버 저장 없음
- 지원 파일 형식
- 브라우저 메모리와 큰 파일 한계
- 사람이 다시 확인해야 하는 지점
- 개인정보 또는 민감정보 주의

특히 파일/녹음/PDF/이미지 도구는 한계 안내를 강화한다.

### 6.4 카테고리 페이지 업무 흐름 보강

카테고리 페이지는 개별 도구 설명을 반복하지 않고, 해당 범주의 업무 흐름을 설명한다.

권장 구성:

- 텍스트 도구: AI 답변 정리, 문서 검수, 목록 정리, 연락처 추출
- PDF 도구: 제출용 PDF 준비, 스캔본 정리, 계약 문서 분리/병합
- 이미지 도구: 업로드 이미지 준비, 사진 개인정보 제거, 웹용 이미지 최적화
- 자막 도구: 자동 자막 정리, 포맷 변환, 싱크 보정
- 음성/영상 도구: 녹음 초안 작성, 녹음 편집, 웹캠 녹화

목표:

- 카테고리 페이지가 단순 링크 모음이 아니라 업무 흐름 안내 페이지로 보이게 한다.

### 6.5 소수의 실무 가이드 작성

대량 블로그는 만들지 않는다. 도구 사용과 직접 연결되는 실무 가이드만 소수 작성한다.

우선 후보:

- PDF 제출 파일을 정리하는 실무 흐름
- 사진 공유 전 EXIF 정보를 지워야 하는 이유
- CSV가 엑셀에서 깨질 때 확인할 것
- AI 답변을 문서에 붙여넣기 좋게 정리하는 법
- 녹음 파일로 회의록 초안을 만들 때 주의할 점

목표:

- 검색용 잡문이 아니라 실제 도구 사용 판단을 돕는 문서로 만든다.

## 7. 구현 원칙

- 홈 화면은 유지한다.
- 도구 작업 UI는 방해하지 않는다.
- 광고 영역은 입력, 업로드, 변환, 결과 UI 안에 넣지 않는다.
- 도구별 문구는 복붙처럼 보이지 않게 작성한다.
- 한국어 작업 후 `/en/`, `/ja/`, `/zh/` 반영 여부를 점검한다.
- `app.js` 또는 `styles.css` 변경 시 정적 리소스 캐시 버전을 올린다.
- 자동 감사에서 콘텐츠 누락을 잡을 수 있는 항목은 감사 스크립트에 추가한다.

## 8. 검증 기준

작업 후 기본 검증:

```bash
npm.cmd run check
git diff --check
```

추가 확인:

- 주요 도구 페이지 200 응답
- 주요 카테고리 페이지 200 응답
- `/en/`, `/ja/`, `/zh/` 주요 도구 페이지 200 응답
- 새 캐시 버전의 `app.js` 또는 `styles.css` 반영 여부
- 도구별 콘텐츠가 화면에서 접히거나 겹치지 않는지 확인
- 같은 문구가 여러 도구에 반복되지 않는지 확인

## 9. AdSense 재검토 전 체크리스트

- 모든 도구에 고유 사용 시나리오가 있는가
- 주요 도구에 입력 예시와 결과 활용 설명이 있는가
- 카테고리 페이지가 업무 흐름을 설명하는가
- 개인정보 처리와 브라우저 처리 원칙이 충분히 설명되는가
- 도구별 한계와 지원 형식 안내가 있는가
- 반복 FAQ나 복붙 문구가 눈에 띄지 않는가
- 광고가 도구 사용을 방해하지 않는가
- 다국어 페이지에 콘텐츠 누락이나 한국어 누출이 없는가
- 운영 URL 확인이 끝났는가
- Search Console 색인 요청 대상 URL 목록을 정리했는가

## 10. 완료 기준

이 개선 작업은 다음 조건을 만족하면 완료로 본다.

- 도구별 고유 시나리오, 입력 예시, 결과 활용, 주의점이 주요 도구에 반영된다.
- 카테고리 페이지가 업무 흐름 중심으로 보강된다.
- 다국어 페이지의 콘텐츠 누락을 자동 감사로 점검한다.
- 배포 후 주요 운영 URL과 캐시 버전 반영을 확인한다.
- Search Console 색인 요청과 AdSense 재검토 요청을 수동으로 진행한다.

## 11. 최종 방향

코워크스페이스의 AdSense 대응은 억지 콘텐츠 추가가 아니라, 기능 중심 플랫폼에 맞는 문서형 콘텐츠 보강으로 진행한다.

핵심 목표는 Google 심사용 글자 수 확보가 아니라, 사용자가 각 도구의 실제 업무 가치를 이해하고 안전하게 사용할 수 있게 만드는 것이다.
