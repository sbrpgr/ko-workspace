# 녹음 파일 간편 편집기 작업 명세

이 문서는 `녹음 파일 간편 편집기` 기능을 다시 수정하거나 이어서 개선할 때 확인해야 할 작업 명세다. 이 기능은 휴대폰 녹음 파일을 서버 업로드 없이 브라우저 안에서 간단히 자르고 붙이는 도구다.

## 기능 목표

- 사용자가 가진 일반적인 휴대폰 녹음 파일을 브라우저에서 열어 파형을 보며 간단히 편집한다.
- 전문 오디오 편집기를 대체하지 않고, 앞뒤 불필요한 구간 삭제, 필요한 구간만 남기기, 짧은 구간 복사·붙이기, 음량 조절처럼 자주 쓰는 작업에 집중한다.
- 원본 녹음 파일, 파일명, 편집 결과, 파형 데이터는 코워크스페이스 서버나 분석 이벤트로 보내지 않는다.
- 편집 결과는 브라우저에서 생성한 WAV 파일로 저장한다.

## 현재 구현 기준

- 도구 ID: `audio-editor`
- URL: `/tools/audio-editor/`
- 렌더러: `app.js`의 `renderAudioEditor`
- 도구 정의: `app.js`의 `TOOL_DEFS` 중 `audio-editor`
- 페이지 파일: `tools/audio-editor/index.html`
- 영어 페이지 파일: `en/tools/audio-editor/index.html`
- 현재 정적 리소스 캐시 버전: `20260510-03`

## 지원 형식 기준

- 목표 지원 형식은 일반적으로 쓰는 휴대폰 녹음 파일로 제한한다.
- 파일 선택 허용값은 `.m4a`, `.aac`, `.mp3`, `.wav`다.
- `amr`, `3gp`, `3ga` 같은 오래되었거나 브라우저 지원이 불안정한 형식은 기본 지원 대상으로 넣지 않는다.
- 같은 확장자라도 내부 코덱이 브라우저에서 디코딩되지 않으면 열리지 않을 수 있다.
- 매우 긴 녹음은 브라우저 메모리 부담이 크므로 실패할 수 있다는 안내를 유지한다.

## UI와 동작 기준

- 파일 선택과 `.upload-box` 드래그앤드롭을 지원한다.
- `canvas`에 파형을 표시한다.
- 파형 드래그로 선택 구간을 만든다.
- 파형 클릭은 붙여넣기나 재생 시작에 쓸 현재 위치를 이동한다.
- 선택 구간이 있으면 선택 재생, 선택 삭제, 선택만 남기기, 선택 복사를 사용할 수 있다.
- 복사한 구간은 현재 위치 또는 선택 시작 지점에 붙인다.
- 실행 취소와 다시 실행을 제공한다.
- 음량 조절은 선택 구간이 있으면 선택 구간에, 없으면 전체 녹음에 적용한다.
- 스페이스바는 편집기에서 재생/멈춤 토글로 동작한다.
- 스페이스바 동작은 버튼, 파일 입력, 슬라이더, select, textarea, 링크, 제목 새로고침 같은 기존 컨트롤의 기본 동작을 방해하지 않아야 한다.

## 출력 기준

- 편집 결과는 mono PCM WAV로 저장한다.
- WAV는 브라우저에서 안정적으로 만들기 위한 기본 출력 형식이다.
- MP3나 M4A 재출력은 브라우저 기본 기능만으로 안정적으로 제공하기 어렵다. 별도 인코더 라이브러리를 검토하기 전에는 기본 출력으로 추가하지 않는다.
- 새 외부 라이브러리를 추가하려면 CDN, CSP, 성능, 개인정보 영향을 함께 검토한다.

## 보안과 개인정보

- 녹음 파일 원본, 파일명, 파형 데이터, 편집 결과를 분석 이벤트나 외부 API로 보내지 않는다.
- 오디오 디코딩과 편집은 `AudioContext`, `Float32Array`, `canvas`를 사용해 브라우저 안에서 처리한다.
- 사용자 파일을 자동으로 외부 링크나 외부 서비스에 전달하지 않는다.
- 오류 문구에는 브라우저 코덱 지원, 파일 형식, 긴 파일 가능성을 안내하되 사용자의 파일명이나 내용은 노출하지 않는다.

## 변경 시 필수 작업

- `app.js` 또는 `styles.css`가 바뀌면 모든 HTML, `site.webmanifest`, `PROJECT_SPEC.md`의 정적 리소스 캐시 버전을 올린다.
- 기능 범위, 지원 형식, 출력 형식, 단축키가 바뀌면 이 문서와 `PROJECT_SPEC.md`, `README.md`, `ROADMAP.md` 중 관련 설명을 맞춘다.
- 파일 입력 ID를 추가하거나 바꾸면 `scripts/project-audit.js`와 `scripts/smoke-test.js`의 드래그앤드롭 허용 목록을 확인한다.
- 편집 순수 함수가 바뀌면 `scripts/smoke-test.js`에 회귀 검사를 유지하거나 추가한다.
- 작업 후 `npm.cmd run check`와 `git diff --check`를 통과시킨다.
- 배포가 필요한 변경이면 커밋, `origin/main` 푸시, GitHub Actions 성공, 운영 URL 반영을 확인한다.
